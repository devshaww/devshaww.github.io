JavaScript二三事

2017-07-19 • 分类: JavaScript

javascript与各浏览器的兼容问题

方法篇
  • IE7及更早版本, unshift(向数组头部插入元素)返回的是undefined, 而不是length. IE8在兼容模式下会返回数组长度.
  • IE7及更早版本在join方法不传参数或者传入undefined时, 会错误地将undefined作为分隔符, 而其他浏览器以,分隔处理; join方法遇到数组中为null, undefined的项, 以空字符串处理.

垃圾回收篇

垃圾回收主要有两种方法: 1) 标记清除 2) 引用计数(会有循环引用问题). JavaScript采用的是标记清除.

  • IE9以前的DOM/BOM元素不是原生js对象, 采用的是引用计数.
  • IE7重写了它臭名昭著的垃圾回收, 性能得到大大的提升.

异常捕获篇
  • 在IE8及之前版本, 在catch语句中捕获的错误对象会被添加到执行环境的变量对象中, 而不是catch语句的变量环境中. 也就是说, 即使是在catch块的外部也可以访问到错误对象, IE9修复了这个问题.

javascript与编程语言传统概念不同的地方

基础篇
  • js的五种基础数据类型: Undefined Null Number Boolean String.
    • 这五种基本数据类型可以通过valueOf()取值. undefined转换成数值是NaN, 而null是0.
    • Undefined Number Boolean String可以通过typeof获得类型, 而对Null以及对象使用typeof会得到Object. 确切的判定对象是否是某类型需要通过instanceof, 即通过原型链来判断.
  • js的Array是可以存不同类型的数据的.
  • js的Array的length不是只读的.
  • js的String不是对象, 是基础数据类型之一.
  • js的sort方法, 若不传参, 会对每个array项调用toString(), 然后比较字符串. 所以一般都需要传入一个函数来定义比较方案.
  • 如果slice()的参数中有一个负数, 则用数组长度加上该数来确定index; 如果结束index小于起始index, 则返回空数组.
  • splice()方法可用于向数组替换、删除、插入元素.
    • 删除: 指定两个参数(起始项和删除的个数) e.g: splice(0,2) -> 删除前两项.
    • 插入: 指定三个参数(起始位置、0(要删除的个数)、要插入的项) e.g: splice(2,0,”red”,”green”) -> 从位置2开始插入元素
    • 替换: 指定三个参数(起始位置、要删除的项数、要插入的项) e.g: splice(2,1,”red”,”green”) -> 删除当前位置2的项, 然后再从位置2开始插入”red”,”green”.

  • js没有块级作用域(即if/while/for不会形成作用域)

  • 用var声明的变量会被添加到最近的环境, 没用var声明但却赋了值的变量, 会被添加到全局作用域. 即: 访问一个没有声明的变量, js会报错; 给一个没有声明的变量赋值, js不会报错, 会被认为是在隐式声明一个全局变量, 但在strict模式下不允许这样.
