CSS Performance

  • Minimize included styles

    • Unused styles increase download size
    • Browser must parse and match all selectors(Failures are expensive)

  • Simplify selectors

    • Complex element selectors are slow
    • when possible
      • Use class - or ID-based selectors
      • Make element selectors as simple as possible
      • Use child instead of descendent selectors
      • Do not mix RTL(Right to Left) and LTR(Left to Right) styles
    • Minimizing included styles makes this easier

  • Don’t use expressions

    • Slow - evaluated frequently

      • 其实我也不知道CSS表达式是什么鬼 查了一下 大概就是在CSS文件里使用函数 动态计算值 形如：

    • Not supported in IE8 standards mood

  • Minimize page re-layouts

    • Poor user experience as content moves
    • Browser performs unnecessary work

JavaScript Coding Ineffeciencies

  • Parsing JSON: Use the native methods
    • JSON.parse()
    • JSON.stringify()
    • toJSON() on prototypes of Date, Number, String and Boolean

  • The switch statement: Use a lookup table(Turn large switch statements into lookup table)

  • Property access methods: Use the direct access(Avoid property access methods)

  • Minimize DOM interaction

  • Smart use of DOM methods

    • nextSibling() better than childNodes[i]

    • querySelectorAll() better for element groups


HTTP Performance

Typical visit

  • HTTP compression: Use it

    • Request Packet: Accept-Encoding:gzip
    • Response Packet: Content-Encoding: gzip

  • Scaled images

  • File linking: Link one CSS file and one JS file

  • Many images: Combine and mask(sprites)


Repeat visits

  • Use Conditional HTTP requests
    • Plain HTTP request
      • pragma: no-cache
    • Time conditional
      • If-modified-since: date,time
  • Provide cacheable content
    • Time conditional
      • Expires: data,time
      • Max-age: #seconds
  • Script blocking
    将javascript脚本放置在body底部 而不是写在head里面
  • Tools
    • Fiddler
      • Inspect network traffic
      • http://www.telerik.com/fiddler
    • neXpert
      • Fiddler plug-in to aid perfomance testing
      • http://www.fiddler2.com/fiddler2/addons/nexPert.asp

HTTP performance takeaways

  • Reduce the number of requests
    • Combine external scripts, styles, and images
    • Use caching
  • Reduce the size of requests
    • Use HTTP compression
    • Use conditional requests
  • Avoid blocking factors
    • Put script at end of HTML
